# 看護師シフト自動生成ツール

2交代制（日勤・夜勤）クリニック規模向け。OR-Tools CP-SAT で 15 種類のハード制約を満たしつつ、勤務日数・夜勤回数・純粋な休の3軸でばらつきを最小化する最適化ソルバ。

## セットアップ

```bash
pip install -r requirements.txt
streamlit run shift_optimizer.py --server.port 8504 --server.fileWatcherType none
```

`--server.fileWatcherType none` は iCloud 上で動かす場合の誤リロード対策（mtime 監視がノイズを拾うため）。

## 機能

- **タブ構成**: 「シフト作成」と「ルール・使い方」の2タブ
- **期間設定**: 年月指定で自動的に日数判定
- **看護師マスタ**: 氏名 / 役割（一般・日勤のみ） / 夜勤可 / 土日休（パートさん向け）
- **希望休**: 看護師ごとに日付カンマ区切り入力
- **制約設定**:
  - 1日あたり 日勤・夜勤の最低人数 / 上限人数
  - 連続勤務上限
  - 月間夜勤 上下限
  - 月間休日 下限・上限（休のみ。明けは別カウント）
  - 月間 最低勤務日数
- **シフト表示**: 色分け表（日=緑 / 夜=紺 / 明=黄 / 休=赤）
- **パターンA / パターンB**: 同条件で別案も同時生成（タブ切替）
- **集計 & CSV出力**
- **INFEASIBLE時の自動診断**: 事前チェック＋制約緩和の詳細診断で原因を特定

## モデル構造

| シフト | 記号 | 意味 |
|--------|------|------|
| 休 | 0 | 完全休日 |
| 日 | 1 | 日勤 |
| 夜 | 2 | 夜勤(翌朝まで) |
| 明 | 3 | 夜勤明け休み |

### ハード制約（15種類）

1. 各人各日ちょうど1シフト
2. 1日あたり日勤・夜勤の最低人員 / 上限人員
3. 夜勤翌日は必ず明け、明けは夜勤翌日のみ
4. **夜→明→休 パターン**: 明けの翌日は必ず休（2連休保証）
5. 「日勤のみ」「夜勤不可」の人員は夜勤なし
6. 「土日休」ON のスタッフは土日固定休
7. 希望休は固定
8. 月間夜勤 上下限
9. 月間休日（休のみ）下限・上限
10. 月間 最低勤務日数（日勤+夜勤）
11. 連続勤務上限

### ソフト制約（目的関数）

```
Z = 3 × 勤務日数の差 + 2 × 夜勤回数の差 + 2 × 休の差
```

を最小化（公平性の最大化）。

## 拡張候補

- 前月末のシフト引き継ぎ（月初の明け反映）
- 既存シフトのロック（部分手動調整）
- 希望休の第1〜第3希望（優先度付きソフト化）
- 有給・研修日
- ペア制約（避けたい/組みたい）
- スキルレベル別ミックス要件
- 土日祝の人員調整（土日のみ最低人員を別設定）
- 看護師マスタの CSV インポート

## 注意点

- **Python 3.14 + OR-Tools 9.15** ではマルチワーカーがハングする既知問題のため、`solver.parameters.num_search_workers = 1` を設定済み。
- 計算の最終状態は `st.session_state` のみで保持。ファイル書き込みは行わないため Streamlit Community Cloud 等にそのまま公開可能。
